Hexo图片部署后无法显示问题

2021-08-31
2021-08-31
hexo, 标签插件

通过常规的 markdown 语法和相对路径来引用其他资源可能会导致他们再存档页或主页上显示不正常。这需要你使用标签插件或者 hexo-renderer-marked 3.1.0 插件以完成对图像的正确引用。

相对路径引用的标签插件

当你打开文章资源文件夹功能后，你把一个 example.jpg图片放在了你的资源文件夹中，如果通过使用相对路径的常规 markdown 语法 ![](example.jpg) ，它将不会出现在首页上。（但是它会在文章中按你期待的方式工作）
如图所示：



在 Hexo 3 中，你可以使用 标签插件 来引用图片和其他资源 ,而不是使用 markdown语言。

通过这种方式，图片将会同时出现在文章和主页以及归档页中。

使用 Markdown 引用图像

hexo-renderer-marked 3.1.0 引入了一个新选项，允许您在不使用 asset_img 标签插件的情况下在 Markdown 中嵌入图像。
启用：

启用后，图像资源将自动解析为相应帖子的路径，并将 ![](image.jpg) 转化为 <img src="/2020/01/02/foo/image.jpg"> 。
使用方法：
启用 post_asset_folder ，在 cmd 或者 bash 中使用命令创建文章，就会自动创建一个同名文件夹，将图片资源放入该文件夹中，正常使用 markdown语言引用该图片相对路径即可。

参考文章：https://hexo.io/zh-cn/docs/asset-folders
